# v1.2 — "Alive Office" design

Approved by Andrea on 2026-07-19 (chat). Ships together with the not-yet-submitted
1.1.0 work as one release; the marketing version stays whatever we submit with.

## Goals (user wishlist)

1. More humor, with on-screen comic text when things happen.
2. Better character movement: richer office life + physical reactions to events.
3. Runs must differ from each other.
4. A task must never repeat within one campaign.
5. Minigames: WarioWare-style micro-gags on "Myself" tasks.

## 1. No-repeat tasks + 12 new tasks (core, TDD)

- `GameState.dealtTaskIDs: [String]`, backward-compatible decode (same pattern
  as `duelsWon`).
- `startDay()` deals only from tasks not yet dealt this campaign and records
  the deal. If the catalog ever runs dry (defensive), it falls back to the full
  catalog rather than dead-ending.
- Worst case is 7 days × 5 tasks = 35 slots, so the catalog grows from 24 to
  ≥36 tasks (~12 new jokes). Count is test-enforced.
- Test: a full campaign never sees the same task twice.

## 2. Run variety via persisted seed (core, TDD)

- `GameState.seed: UInt64`, set at campaign start, backward-compatible decode
  (legacy saves get a random seed once).
- The resume `GameEngine` init drops its `seed:` parameter and uses
  `state.seed`. The daily challenge just stores `DailySeed.seed()` here —
  resume determinism comes free and the App-side re-seeding hack goes away.
- Duels and consultant offers are shuffled once per campaign with a dedicated
  `SeededRandomNumberGenerator(seed: state.seed)` (separate from the day RNG so
  the order is stable no matter how much of the campaign has run).
  `duelForToday()` / `consultantForTonight()` index into the shuffled arrays.
- Tests: same seed ⇒ same duel order across resumes; different seeds ⇒
  different order (spot-checked); legacy decode.

## 3. Comic text in the scene ("scritte")

- `OfficeScene.spawnComicText(_ text: String, at: CGPoint)`: pixel-font
  SKLabelNode with outline, scale-in pop, wobble, drift up, fade out.
- Data-driven where content-specific: optional `sting` string on
  `OfficeEvent` (`events.json`) — e.g. "GINO È LIBERO!", "BZZT." Shown at the
  event prop's position when the event fires. All 15 events get one.
- Generic pools for task resolutions (human: "YAY!", "FATTO A MANO",
  ai: "BZZT!", "AUTOMATED.") and duel rounds ("POW!", "OUCH!").
- No full-screen banners.

## 4. Office life + event reactions (scene only, no new art)

- Per-NPC routine state machine on random timers: coffee-machine break,
  two-person chat huddle (with "..." bubble), sitting at a desk, Gino checking
  his mug.
- On event trigger, a physical reaction keyed by event: converge on the new
  prop and look at it, flee from the drone, applaud the barista's return.
  Fallback reaction: everyone briefly turns toward the prop.
- Existing sprites only (walk/react sheets already sliced at runtime).

## 5. Micro-gags (UI + one core hook)

- `OfficeTask.microGame: String?` in `tasks.json` (nil = no minigame).
  Launch set, matched to existing tasks:
  - `printer_smash` on *fix_printer* — tap the printer N times in 3 s.
  - `coffee_rush` on *coffee_run* — stop a moving bar in the sweet zone, 3×.
  - `find_lasagna` on *excel_cleanup* — tap the 'lasagna' cell in a grid
    before the timer runs out.
- Flow: choosing "🙋 Myself" on such a task presents the micro-game overlay
  first; the task then resolves normally. Choosing "🤖 The AI" always skips
  the minigame (delegating means not playing — that's the theme).
- Success: engine applies a flat bonus `Consequence` (+2 humanity, no
  automation change) — core hook `resolveMicroGameBonus()`, test-covered —
  and the UI shows a triumphant line. Failure: no penalty, alternate gag line.
  Comedy first: never punish.
- Micro-game views are plain SwiftUI with timers; each always terminates
  within ~5 s so the campaign UI test can button-mash through them.

## Testing

- Core: strict TDD for §1, §2, §5 hook (RED first, as always).
- Scene/UI: full-campaign UI test updated for micro-game overlays; simulator
  screenshots via the `testShowcase` harness for comic text and office life.

## Out of scope

New sprite art, standalone minigames, three-act structure, iPad.
