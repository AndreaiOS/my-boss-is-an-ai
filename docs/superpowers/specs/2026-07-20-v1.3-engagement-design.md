# v1.3 — "Engagement & polish" design

Approved by Andrea on 2026-07-20 (chat). Four self-contained features shipped
together. Boss-in-scene presence is deferred (waiting on new sprite art); the
authored three-act structure is a separate future design.

## Goals

1. Share card — a shareable image of the campaign's ending (viral loop).
2. Daily streak — reward consecutive daily-challenge days.
3. Notebook — a collection screen: endings found, learned comebacks, fun stats.
4. Adaptive music — chiptune loops per office stage, colder as automation rises.

## 1. Share card

- End screen (`endingOverlay`) gains a "Share ▸" button beside "Play again".
- `ShareCardContent` (core, pure, testable): built from `Ending`, `OfficeState`,
  `duelsWon`, and optional daily score. Produces the display strings:
  `title` (ending title upper-cased), `tagline` (ending flavor, trimmed to one
  line ≤ 90 chars ending on a word), `scoreLine` ("🤖 62  ❤️ 40"), and
  `footer` ("My Boss Is an AI"). Daily runs append " · ☀️ 275".
- `ShareCardView` (App, SwiftUI): 1080×1080 pixel-styled card — ending artwork
  banner, title, tagline, score line, wordmark. Rendered to `UIImage` via
  `ImageRenderer` (scale 1, fixed 1080 frame) and handed to
  `UIActivityViewController`.
- No impersonation: it's the player's own result. In-app iOS share sheet only,
  no Artifact.
- Tests: `ShareCardContent` string formatting (tagline truncation on a word
  boundary; daily vs non-daily score line). Card render verified by simulator
  screenshot.

## 2. Daily streak

- `DailyStreak` (core, pure, TDD): `update(streak:lastDay:today:)` →
  `(streak: Int, lastDay: DateComponents-as-Int)`. Day key is the UTC
  `year*10000+month*100+day` integer (same convention as `DailySeed`).
  Rules: same day key ⇒ unchanged; key exactly one calendar day later ⇒ +1;
  any larger gap ⇒ reset to 1. Uses a UTC gregorian calendar to compare.
- App wiring: on daily-challenge completion (`reportToGameCenter`, inside the
  `DailyChallenge.isActiveToday` branch) update stored `dailyStreak` and
  `dailyStreakLastDay` in UserDefaults, and submit the streak to GC leaderboard
  `daily_streak` (best = highest). Show "🔥 N-day streak" on the daily end card.
- ASC: new recurring? No — a classic all-time leaderboard `daily_streak`
  (best = highest). Added to `docs/appstore/checklist.md`.

## 3. Notebook

- New `NotebookView` reached from a title-screen button "📓 Notebook".
- Sections: endings (thumbnail grid, X/7, unfound shown as a "？" silhouette
  card using the ending art tinted/darkened), learned comebacks count with ★,
  and fun stats.
- Stats: a small `Stats` enum (App) wrapping UserDefaults integer counters,
  with `increment(_:)` / `value(_:)`. Counters: `campaignsCompleted` (exists),
  `duelsWonTotal`, `microGamesWon`, `printersSmashed` (a fun alias for
  micro-games won on the printer — keep to `microGamesWon` only to avoid
  double-counting; expose it in copy as "printers persuaded"). Wire increments
  in the VM: `duelsWonTotal` when a bout is won, `microGamesWon` in
  `finishMicroGame(won: true)`.
- `NotebookContent` (core, pure, testable): given the endings catalog and the
  set of found ids, returns `[NotebookEnding(id, title, found)]` in catalog
  order. Keeps the found/locked mapping out of the view.
- Tests: `NotebookContent` maps found vs locked correctly and preserves order.

## 4. Adaptive music

- `tools/make_sfx.py` extended (new `make_music()` or sibling script) to
  synthesize three seamless ~8 s looping tracks written to
  `App/Resources/`: `music_lively`, `music_hybrid`, `music_automated`.
  Warmth decreases with automation: lively = triad arpeggio + swing, hybrid =
  straighter + one voice dropped, automated = single square-wave metronome
  pulse. WAV/CAF, mono, 22–44 kHz, tuned for seamless loop (whole number of
  bars).
- `MusicPlayer` (App): holds three looping `AVAudioPlayer`s; `setStage(_:)`
  crossfades (volume ramp ~1 s) to the track for the current stage; respects a
  new `Settings.musicOn` toggle (default on) added to the options sheets.
  Started when a game scene appears, stopped on exit.
- `MusicTrack(for: OfficeStage)` — pure mapping, testable in App (or a tiny
  core enum). Audio itself verified by ear.
- `GameView` calls `MusicPlayer.setStage` wherever `syncScene()` already runs
  (day boundaries), so music shifts with the visible office.
